1. Ansible 是什麼？

開源自動化工具

由 Red Hat 維護，擁有龐大且活躍的社群，透明度高且免費使用。

無代理架構 (Agentless)

最大特色！僅透過標準 SSH 或 WinRM 通訊，無需在受管節點安裝任何代理程式。

YAML Playbook

使用人類可讀的 YAML 語言來描述自動化流程，語法簡單直觀。

基礎設施即程式碼 (IaC)

將基礎設施組態用程式碼管理，實現版本控制、可重複性與可追蹤性。

2. 架構與核心概念

(Control Node)

安裝 Ansible 的主機

ansible-playbook ...

SSH / WinRM

受管節點 1

(Web Server)

受管節點 2

(Database)

受管節點 N

(...)

Inventory: 定義受管主機的清單檔案。

Modules: 執行特定任務的工具 (如 apt, copy)。

Tasks: 一個動作，即是呼叫一個 Module。

Playbook: 由多個 Tasks 組成的自動化劇本。

3. 安裝與設定

安裝 Ansible

# Ubuntu / Debian
sudo apt install -y ansible

# CentOS / RHEL
sudo yum install -y ansible

設定 SSH 金鑰認證 (關鍵！)

讓 Ansible 可無密碼登入受管主機。

# 1. 在控制節點生成金鑰 (若無)
ssh-keygen

# 2. 將公鑰複製到受管節點
ssh-copy-id your_user@managed_node_ip

4. Ad-hoc 命令

適用於執行快速、一次性的臨時操作。

測試所有主機連線:

ansible all -m ping

檢查 webservers 群組的開機時間:

ansible webservers -m shell -a "uptime"

確保 webservers 的 nginx 服務已啟動 (需 root 權限):

ansible webservers -m service -a "name=nginx state=started" --become

5. Playbook 詳解

自動化的核心劇本，使用 YAML 格式，具備 冪等性 (Idempotency) 特性。

範例: `deploy_nginx.yml`

---
- name: Deploy and configure Nginx
  hosts: webservers
  become: yes
  tasks:
    - name: Install Nginx package
      apt:
        name: nginx
        state: latest

    - name: Ensure Nginx is running
      service:
        name: nginx
        state: started
        enabled: yes

如何執行:

ansible-playbook -i inventory.ini deploy_nginx.yml

7. 實戰：部署 K3s 叢集

1. 定義 Inventory

[k3s_server]
master1 ansible_host=192.168.56.10

[k3s_agents]
agent1 ansible_host=192.168.56.11

2. 撰寫 Playbook (部分)

- name: Install K3s Server
  hosts: k3s_server
  become: yes
  tasks:
    - name: Run K3s installer
      shell: curl -sfL https://get.k3s.io | sh -
      args:
        creates: /usr/local/bin/k3s
    - name: Get the server token
      shell: cat /var/lib/rancher/k3s/server/node-token
      register: server_token

3. 驗證叢集

ansible k3s_server -m shell -a "sudo kubectl get nodes"

8. 最佳實踐

  • ✓ 保持冪等性: 盡量使用內建模組，謹慎使用 `shell`。
  • ✓ 使用 Roles 模組化: 將複雜的 Playbook 拆分為可重用的角色。
  • ✓ 使用 Ansible Vault 保護機密: 絕不將密碼、金鑰明文儲存。
  • ✓ 整合 CI/CD: 將 Ansible 納入自動化部署流程 (Jenkins, GitLab CI)。
